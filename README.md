# Customer Behavior Analysis & SQL Case Study

## Overview

This project focuses on analyzing customer behavior using SQL, Python, and Power BI. The main aim was to understand how customers interact with a business, identify patterns in their purchasing habits, and generate insights that can help in better decision-making.

The project covers data cleaning, feature engineering, SQL-based analysis, and dashboard creation.

---

## Tools Used

* PostgreSQL (pgAdmin)
* Python (Pandas, NumPy, Matplotlib)
* Power BI

---

## Project Files

* Customer_Behavior_sql_queries.sql → contains all SQL queries used for analysis
* Customer_Behaviour_Analysis.ipynb → Python notebook for data cleaning and preprocessing
* Customer_behavior_dashboard.pbix → Power BI dashboard file

---

## Work Done

### Data Cleaning (Python)

* Handled missing values in columns like review rating using category-wise median
* Converted categorical data into numerical form (e.g., purchase frequency into days)
* Created new features like age groups using quantile-based segmentation

---

### SQL Analysis

Solved 10 business-related problems using SQL, such as:

* Total revenue and sales trends
* Top customers based on spending
* Customer distribution across categories
* Purchase patterns and frequency analysis

Used SQL concepts like:

* GROUP BY, ORDER BY
* Aggregation functions (SUM, COUNT, AVG)
* Joins and filtering

---

### Dashboard (Power BI)

Created an interactive dashboard showing:

* Overall revenue and customer metrics
* Customer segmentation
* Purchase behavior trends
* Key patterns in the data

---

## Key Insights

* A small group of customers contributes to a large share of revenue
* Customer purchase frequency varies significantly
* Some categories show more consistent engagement than others
* Data can be used to target high-value customers more effectively
